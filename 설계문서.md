# AI CLI 통합 설치 마법사 설계 문서

## 1. 개요
### 1.1 목적
Windows 환경에서 여러 AI CLI 도구(Codex, Claude Code 등)를 단일 WPF 설치 마법사로 설치, 계정 연결, 공통 규칙(RuleSet) 적용까지 일관되게 관리한다.

### 1.2 목표
- AI CLI 도입 장벽을 낮춘다.
- 공통 RuleSet으로 모델 간 동작 일관성을 확보한다.
- 신규 AI 모델/CLI를 최소 변경으로 확장할 수 있는 구조를 제공한다.

## 2. 대상 사용자
- Windows 개발자
- Codex, Claude Code 등 AI CLI 사용자
- 여러 AI 모델에 동일 규칙을 적용하고 싶은 사용자

## 3. 요구사항
### 3.1 기능 요구사항
- AI CLI 설치 마법사 제공
- 1차 지원 대상: Codex, Claude Code
- 신규 모델/CLI 추가 가능한 확장형 아키텍처
- 모델별 계정 연결 가이드(API Key, OAuth 등)
- 모든 CLI에 적용 가능한 공통 RuleSet 설정/관리
- 기존 설치 환경 탐지 및 존중

### 3.2 비기능 요구사항
- Windows에서 안정적으로 실행
- WPF 기반(.NET 8)
- 관리자 권한 요청 UX 명확화
- 실패 시 원인 진단과 복구 가이드 제공

## 4. 시스템 아키텍처
다음 모듈로 구성한다.

- WPF Installer Application
- CLI Detection Module
- CLI Installation Manager
- Account Connection Wizard
- RuleSet Manager
- Model Registry(확장 포인트)
- Execution & Validation Layer

## 5. 설치 마법사 흐름
### 5.1 진입 화면
- 웰컴 화면 표시
- 지원 AI 모델 목록 표시
- 기존 CLI 설치 상태 자동 탐지

### 5.2 CLI 설치 탐지
탐지 기준:
- PATH 환경 변수
- 기본 설치 경로
- 실행 테스트(예: `--version`)
- 설정 파일 존재 여부

탐지 결과 분류:
- Not Installed: 미설치, 설치 필요
- Installed: 설치됨, 계정 연결 단계 진행
- Installed and Connected: 설치+연결 완료, 설정 관리 중심
- Invalid/Broken: 손상 상태, 복구 또는 재설치 권장

### 5.3 설치 단계
- CLI별 설치 전략을 독립 구현
- 이미 설치된 경우 강제 재설치 금지
- 사용자 선택 기반 설치/갱신/유지

## 6. 모델 계정 연결
### 6.1 지원 방식
- API Key 입력
- OAuth 기반 웹 인증
- CLI 인증 명령 실행

### 6.2 UX 원칙
- `지금 연결` 또는 `나중에 연결` 선택 가능
- 연결 성공 여부 검증
- 간단한 테스트 실행 제공

## 7. 공통 RuleSet 관리
### 7.1 RuleSet 개념
모든 AI CLI에 공통 적용되는 규칙 집합을 제공한다.

예시:
- 추가 질문 금지
- 특정 디렉터리 접근 허용
- 작업 완료 후 문서화 강제
- WPF/MVVM 개발 규칙 강제

### 7.2 저장 구조
애플리케이션 데이터 경로 하위 예시:

```text
rulesets/
  default.ruleset.json
  custom.ruleset.json
  model-mapping.json
```

### 7.3 적용 방식
- CLI 프롬프트 또는 설정 파일에 자동 주입
- 모델별 override 허용
- 사용자 직접 편집 가능

## 8. 확장 가능한 모델 아키텍처
### 8.1 Model Registry
모델 정의에는 다음 정보를 포함한다.
- Model Identifier
- Display Name
- Installation Strategy
- Authentication Strategy
- RuleSet 지원 여부

### 8.2 신규 모델 추가 절차
1. Registry에 모델 정의 추가
2. 설치/인증 전략 구현
3. UI에서 자동 노출 및 흐름 반영

## 9. 기존 설치 환경 처리 정책
### 9.1 원칙
- 기존 사용자 환경을 존중
- 강제 덮어쓰기 금지
- 사용자 선택 우선

### 9.2 제공 옵션
- 기존 CLI 그대로 사용
- 설정만 관리
- 최신 버전 마이그레이션
- 사이드 바이 사이드 설치

## 10. 기술 스택
- UI: WPF
- Runtime: .NET 8
- Architecture: MVVM
- Configuration: JSON
- Process 실행: `System.Diagnostics.Process`
