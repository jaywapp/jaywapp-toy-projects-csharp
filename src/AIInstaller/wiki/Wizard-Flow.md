# 마법사 흐름

## 전체 흐름

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Step 1      │    │  Step 2      │    │  Step 3      │    │  Step 4      │
│  상태 점검    │───▶│  설치 진행    │───▶│  규칙 설정    │───▶│  완료        │
│              │    │              │    │              │    │              │
│  [DataGrid]  │    │  [DataGrid]  │    │ [RuleEditor] │    │  [DataGrid]  │
└──────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
```

## Step 1: 상태 점검

**제목**: 설치 상태 점검
**설명**: 시스템에 설치된 AI CLI 도구의 현재 상태를 점검합니다.

### 탐지 프로세스

각 CLI에 대해 다음 순서로 탐지합니다:

1. **PATH 확인**: `where <command>` (Windows) 실행
2. **버전 확인**: `<command> --version` 실행
3. **설정 파일 확인**: 계정 설정 파일 존재 여부 검사

### 탐지 결과 상태

| 상태 | 표시 | 의미 |
|------|------|------|
| `NotInstalled` | ❌ 미설치 | CLI가 시스템에 없음 |
| `Installed` | ⚠️ 설치됨 (계정 미연결) | CLI는 있지만 계정 설정 파일이 없음 |
| `InstalledAndConnected` | ✅ 설치 및 연결 완료 | CLI 설치 + 계정 설정 완료 |
| `InvalidOrBroken` | ❌ 설치 손상 | CLI가 존재하나 `--version` 실행 실패 |

### 권장 조치

| 상태 | 권장 조치 |
|------|----------|
| `NotInstalled` | 🔧 설치 필요 |
| `Installed` | 🔗 계정 연결 필요 |
| `InstalledAndConnected` | ✅ 완료 |
| `InvalidOrBroken` | 🔧 재설치 필요 |

**다음 버튼 클릭 시**: 상태 점검을 다시 실행하고 Step 2로 이동

## Step 2: 설치 진행

**제목**: 미설치 항목 설치
**설명**: 미설치 또는 손상된 CLI 도구를 자동으로 설치합니다.

### 설치 대상

`NotInstalled` 또는 `InvalidOrBroken` 상태인 CLI만 자동 설치 대상입니다.

### 설치 방법

| CLI | 설치 명령 |
|-----|----------|
| Claude Code | `npm install -g @anthropic-ai/claude-code` |
| Codex | `npm install -g @openai/codex` (실패 시 `npm install -g codex-cli` 폴백) |

### 설치 흐름

1. 설치 명령 실행 (`ProcessCommandRunner`)
2. 설치 후 `<command> --version`으로 검증
3. 성공/실패 결과를 모델별로 수집
4. 상태 다시 점검 (DataGrid 갱신)

**다음 버튼 클릭 시**: 미설치 항목 설치 후 RULE.md를 로드하고 Step 3으로 이동

## Step 3: 규칙 설정

**제목**: 전역 규칙 설정
**설명**: RULE.md를 편집하면 CLAUDE.md와 AGENTS.md에 참조가 자동 생성됩니다.

### UI 구성

이 단계에서는 DataGrid 대신 규칙 편집기가 표시됩니다:

- **대상 디렉토리**: RULE.md가 저장될 경로 (자동 탐지 또는 수동 입력)
- **RULE.md 내용**: 모노스페이스 텍스트 에디터 (Consolas 폰트)

### 디렉토리 자동 탐지

앱 실행 경로에서 상위 디렉토리를 순회하며 CLAUDE.md, AGENTS.md, RULE.md 중 하나라도 존재하는 디렉토리를 찾습니다. 없으면 사용자 홈 디렉토리(`%USERPROFILE%`)를 기본값으로 사용합니다.

### 기본 템플릿

RULE.md가 존재하지 않으면 다음 기본 템플릿이 제공됩니다:

```markdown
# 공통 AI 코딩 규칙

## 작업 권한
- 작업 디렉토리 및 모든 하위 디렉토리에 대한 전체 권한이 사전 승인됨
- 해당 범위 내에서 접근 확인을 재차 요청하지 않는다

## 자율 실행
- 작업 수행 중 사용자에게 질문하지 않는다
- 합리적인 판단으로 자율적으로 진행한다

## 비파괴적 기본 원칙
- 명시적으로 필요하지 않은 한 파괴적 작업을 피한다
- 기존 파일 수정 시 점진적 변경을 선호한다

## 완료 지향
- 모든 요청 기능이 구현되고 빌드 성공할 때까지 완료로 간주하지 않는다

## 언어
- 기본적으로 한국어로 응답한다
```

**다음 버튼 클릭 시**: RULE.md 저장 + CLAUDE.md/AGENTS.md 참조 자동 생성 + 상태 재점검 후 Step 4로 이동

## Step 4: 완료

**제목**: 설치 완료
**설명**: 모든 설치 작업이 완료되었습니다. 최종 상태를 확인하세요.

최종 상태 DataGrid가 다시 표시되며, 모든 CLI의 현재 상태를 확인할 수 있습니다.

- **완료 버튼**: 상태를 한 번 더 갱신하고 완료 메시지 표시
- **종료 버튼**: 애플리케이션 종료

## 네비게이션

| 버튼 | 조건 | 동작 |
|------|------|------|
| 이전 | Step > 1 | 이전 단계로 이동 |
| 다음 | Step < 4 | 현재 단계 작업 실행 후 다음 단계로 이동 |
| 완료 | Step == 4 | 최종 상태 갱신 |
| 종료 | 항상 | 앱 종료 |

모든 버튼은 `IsBusy == true`일 때 비활성화됩니다.

## 상태 바

하단 상태 바에 현재 작업 상태가 이모지와 함께 표시됩니다:

| 이모지 | 의미 | 예시 |
|--------|------|------|
| ⏳ | 진행 중 | ⏳ 설치 상태를 점검 중입니다... |
| ✅ | 성공 | ✅ 점검 완료. 미설치 항목 설치를 진행하세요. |
| ❌ | 실패 | ❌ 설치 실패: npm not found |
| ⬅️ | 이전 이동 | ⬅️ 이전 단계로 이동했습니다. |
